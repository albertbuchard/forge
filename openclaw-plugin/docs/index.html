Forge

One workspace for work, memory, and review.

Forge keeps goals, projects, tasks, live work, wiki pages, notes, sleep, sports, preferences, and Psyche records in one system. OpenClaw and Hermes can read it, update it, and hand off to the UI when needed.

Planning

Goals, projects, tasks, habits, strategies, notes, and calendar records stay linked.

Execution

Task runs track real work. Work adjustments correct minutes without pretending a run happened.

Wiki memory

The wiki keeps durable pages, backlinks, search, health checks, and background ingest.

Health

Sleep and sports are first-class review surfaces with notes, tags, and Forge links.

Preferences

The app includes a dedicated preferences surface with comparison flows and concept libraries.

Psyche

Values, beliefs, patterns, modes, and trigger reports stay in the same system as the work.

Choose A Surface

OpenClaw, Hermes, or standalone.

Use the adapter that matches how you want to run Forge.

OpenClaw

Published plugin. Best if you want the standard install path and the Forge runtime commands in OpenClaw.

Hermes

Native Python plugin. Best if Hermes is your main agent runtime.

Standalone

Run Forge directly from the repo if you only need the app and the API.

Getting Started

Install Forge in OpenClaw.

This is the fastest path for a normal local install.

Setup

1. Install

Install the published OpenClaw plugin and enable it.

2. Start

Restart the gateway and run the Forge health check.

3. Open

Open the Forge UI or ask the agent to use the tools directly.

Default local UI

The common local UI address is http://127.0.0.1:4317/forge/.

Default data path

OpenClaw stores Forge under ~/.openclaw/extensions/forge-openclaw-plugin/data/forge.sqlite unless you set a custom dataRoot.
