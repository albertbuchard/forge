Forge For OpenClaw

One place to run your goals, work, notes, and reflection.

Forge is a local-first planning and execution system. It gives OpenClaw a real place to store long-term goals, active projects, concrete tasks, live work sessions, linked notes, weekly review, and Psyche records such as values, beliefs, patterns, and trigger reports.

If you want one serious system instead of scattered notes and half-finished task lists, Forge is the operating layer. Install the plugin, allow it, restart OpenClaw, and check that the runtime is actually up.

What Forge Is

Forge is not just a task list.

Plan with real structure

Forge stores goals, projects, and tasks as connected records so you can see why a task matters instead of rebuilding that context in your head.

Track real work

Work sessions are real runs with real time, not fake checkboxes. OpenClaw can start, focus, complete, and review them through the plugin.

Keep useful notes

Notes are first-class Markdown records. They can link to goals, projects, tasks, and Psyche records so progress, evidence, and handoff context stay attached to the work.

Reflect without losing the plot

Psyche records help you track values, beliefs, modes, patterns, and trigger reports without separating reflection from the rest of your actual life and work system.

Getting Started

Use this install flow.

This is the reliable local setup. The important part is that enable is not enough on its own. After enabling the plugin, make sure it is in the allow list, then restart OpenClaw and verify that Forge really started.

Install Forge on a local OpenClaw machine 
openclaw plugins install forge-openclaw-plugin
openclaw plugins enable forge-openclaw-plugin
node -e 'const fs=require("fs"); const p=process.env.HOME+"/.openclaw/openclaw.json"; const j=JSON.parse(fs.readFileSync(p,"utf8")); j.plugins ??= {}; j.plugins.allow = Array.from(new Set([...(j.plugins.allow || []), "forge-openclaw-plugin"])); fs.writeFileSync(p, JSON.stringify(j, null, 2)+"\n");'
openclaw gateway restart
openclaw forge health
openclaw forge status
1

Install the plugin

Install the published package, then enable it in OpenClaw.

2

Repair the allow list

Some OpenClaw builds leave the plugin enabled but not allowed. The Node command above appends forge-openclaw-plugin without breaking the rest of your allow list.

3

Restart OpenClaw

Restart the gateway so the plugin entrypoint and the local Forge runtime load for real.

4

Verify the runtime

openclaw forge health checks the server. openclaw forge status tells you whether Forge is running, healthy, and managed by the plugin.

Open the UI

Ask your OpenClaw agent to open Forge or give you the local address. The normal local UI address is usually http://127.0.0.1:4317/forge/.

If 4317 is already taken, Forge can move to the next free local port and remember that choice for future runs.

Where the data lives

On a normal install, Forge usually stores its SQLite data under ~/.openclaw/extensions/forge-openclaw-plugin/data/forge.sqlite.

If you want the data somewhere else, set dataRoot in the plugin config and restart the gateway.

If startup fails

Check the runtime log at ~/.openclaw/logs/forge-openclaw-plugin/<host>-<port>.log.

That log captures the Forge child process output, which is the fastest way to see port errors, missing dependencies, or runtime crashes.

What It Looks Like

The product has both an execution side and a reflection side.

What Forge Stores

These are the main record types you work with.

Forge keeps the naming plain. A goal is a goal. A project is a project. A task is a task. The plugin exposes these records to OpenClaw so the agent can read, create, update, and manage them without inventing a second system.

Goals

Long-term directions you care about, such as health, family, money, or creative work.

Projects

Concrete bodies of work that move one goal forward.

Tasks

Specific actions inside projects that can be scheduled, moved, started, and completed.

Task Runs

Real work sessions with real time, focus state, and completion history.

Notes

Markdown notes that can attach to one or many records as evidence, handoff, or context.

Insights

Saved observations or recommendations from you or a trusted agent.

Values

Important directions or principles you want your actions to reflect.

Beliefs

Explicit belief sentences that shape behavior, for better or worse.

Patterns

Repeating loops with triggers, actions, short-term payoff, and long-term cost.

Trigger Reports

Structured records of emotionally important episodes, including what happened and what followed.

Real Examples

What using Forge actually looks like.

These are the normal kinds of interactions Forge is built for. The point is not just to store records. The point is to help you decide what matters, work on it, and keep the evidence connected.

Goals and projects

Turn a direction into real work

You create a goal like Build a stable, healthy body. Under it, you create a project like Spring training block. Inside that project, you add tasks like Book Tuesday gym session, Plan meal prep, and Track sleep for 7 days.

Forge keeps the whole chain visible, so when you look at a task you can still see the project and goal it belongs to.

Kanban tasks

Run your work on a board that means something

You keep tasks in columns like backlog, focus, in progress, blocked, and done. A task such as Write plugin install guide can start in backlog, move to focus when it matters today, shift to in progress once work begins, and land in done when the write-up is actually finished.

The board is not just a pretty view. It is the main execution surface for the project.

Live work

Track real sessions, not just finished tasks

You start a task run on Refactor Forge install docs. Forge records when the session started, keeps it visible as your current work, and lets OpenClaw focus, release, or complete the run.

That means the system can tell the difference between a task that exists and work that truly happened.

XP and review

Make progress visible

When you start work, keep going, and complete meaningful tasks, Forge records XP and shows the reward trail in weekly review. Instead of a vague feeling that you were busy, you can see the work sessions, completions, and momentum that actually happened.

Notes

Keep useful context attached to the work

After finishing Write plugin install guide, you save a note with what was changed, what still feels risky, and which commands were tested. That note can stay linked to the task, the project, and the wider goal instead of living in a random text file.

Psyche

Track patterns that affect the work

You notice a repeating loop: when release steps feel messy, you delay them, then feel shame, then avoid the task even more. In Forge, you can store that as a pattern, link it to a belief like If I get one release step wrong, the whole thing is a disaster, and attach a trigger report when a real episode happens.

That keeps reflection grounded in the same system as your goals and tasks.

FAQ

Common problems, after the main setup is already clear.

Forge installed, but nothing starts

The most common cause is that the plugin was installed and enabled, but not added to plugins.allow. Run the full install flow above, especially the allow-list repair command, then restart the gateway.

Why is Forge not on port 4317?

If something else already uses 4317, Forge can move to the next free local port. Use openclaw forge status to see the actual base URL it chose.

Can I store the data somewhere else?

Yes. Set dataRoot in the Forge plugin config, restart the gateway, and Forge will use that folder instead of its default runtime path.

Where do I go if the runtime crashes?

Read the runtime log at ~/.openclaw/logs/forge-openclaw-plugin/<host>-<port>.log. It captures the child process output from the actual Forge server.
