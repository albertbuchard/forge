Forge OpenClaw Plugin

Install Forge cleanly, allow it, and verify the local runtime actually came up.

Forge gives OpenClaw one place for goals, projects, tasks, notes, live work sessions, weekly review, and Psyche records. The reliable local flow is: install the plugin, make sure it is allowed and enabled, restart the gateway, then verify Forge with a real command.

Install from npm GitHub repo
If Forge installs but does not run

The usual cause is that the plugin is installed but not yet active in OpenClaw config. If you see an empty plugins.allow problem, or Forge never starts on localhost, check the allow list directly after install instead of assuming the plugin was fully activated.

Want to open Forge itself?

Ask your OpenClaw agent to open the Forge UI or to give you the local Forge address. The usual local address is http://127.0.0.1:4317/forge/.

Example prompts: Open the Forge UI, Give me the Forge UI address, Take me to Forge

Where the data lives by default

On a normal OpenClaw install, Forge usually stores its SQLite data at ~/.openclaw/extensions/forge-openclaw-plugin/data/forge.sqlite. If you want the data somewhere else, set dataRoot in the plugin config and restart the gateway.

Reliable local install flow 
openclaw plugins install forge-openclaw-plugin
openclaw plugins enable forge-openclaw-plugin
node -e 'const fs=require("fs"); const p=process.env.HOME+"/.openclaw/openclaw.json"; const j=JSON.parse(fs.readFileSync(p,"utf8")); j.plugins ??= {}; j.plugins.allow = Array.from(new Set([...(j.plugins.allow || []), "forge-openclaw-plugin"])); fs.writeFileSync(p, JSON.stringify(j, null, 2)+"\n");'
openclaw gateway restart
openclaw forge health
openclaw forge status
What the verify step does

openclaw forge health confirms that the plugin entrypoint is live and that the local Forge server is actually answering on 127.0.0.1:4317. openclaw forge status tells you whether the runtime is plugin-managed or something you started manually elsewhere.

Always repair the allow list after enable on flaky installs

openclaw plugins enable forge-openclaw-plugin only marks the plugin enabled. If your OpenClaw build keeps leaving plugins.allow empty or stale, run the exact repair command above right after enable. It preserves the rest of the allow list and appends forge-openclaw-plugin if it is missing.

Manage the plugin-managed local runtime cleanly

Run openclaw forge start, openclaw forge stop, openclaw forge restart, or openclaw forge status to manage a Forge runtime that OpenClaw auto-started itself. If Forge was started some other way, the commands explain that instead of killing unrelated local processes.

What This Is

A real planning, execution, and reflection system for OpenClaw.

Forge is one place for long-term goals, active projects, concrete tasks, honest live work sessions, weekly review, XP feedback, and structured Psyche records.

The plugin gives the agent the parts it actually needs: read current context, search records, create or update records, control live work sessions, post insights, and hand the user off to the UI when the visual workflow is easier.

On localhost targets, the plugin can auto-start the bundled Forge runtime. That keeps installation simple: install the plugin, restart OpenClaw, and point it at 127.0.0.1:4317.

Real Install Flow

Install it, allow it, restart, and verify that Forge is really up.

1. Install

openclaw plugins install forge-openclaw-plugin
openclaw plugins enable forge-openclaw-plugin

2. Repair allow list

node -e 'const fs=require("fs"); const p=process.env.HOME+"/.openclaw/openclaw.json"; const j=JSON.parse(fs.readFileSync(p,"utf8")); j.plugins ??= {}; j.plugins.allow = Array.from(new Set([...(j.plugins.allow || []), "forge-openclaw-plugin"])); fs.writeFileSync(p, JSON.stringify(j, null, 2)+"\n");'

3. Restart

openclaw gateway restart

4. Verify

openclaw forge health
openclaw forge status

5. Check the result

openclaw config get plugins.allow --json

6. Open Forge

Ask your agent: Open the Forge UI or Give me the Forge UI address.

7. Runtime controls

Use openclaw forge start, stop, restart, and status for the plugin-managed local runtime.

8. Change data folder

Set plugins.entries["forge-openclaw-plugin"].config.dataRoot if you want the data outside the default install path.

Product Surfaces

Two main surfaces: work management and Psyche mapping.

The overview keeps goals, current work, weekly XP, and progress visible. The Psyche graph shows how values, beliefs, projects, and goals connect, so the user does not have to keep these links in scattered notes. Forge also treats Markdown notes as first-class linked records, so progress evidence and handoff context stay searchable across the product.

Overview Momentum, streaks, live work, and active priorities in one shell.
Psyche Values, beliefs, modes, and trigger chains connected instead of scattered.
Execution Real task runs, durable linked notes, and review instead of fake status toggles.

Core Entities

These are the main record types Forge stores.

goal

A long-term direction or outcome, such as building a stable family or shipping meaningful work.

project

A bounded piece of work under a goal, such as launching the Forge plugin or planning a move.

task

A concrete action or deliverable, such as drafting a README or calling a landlord.

task_run

One real live work session on a task. It is the truthful record of work happening now.

insight

An agent-written observation, recommendation, or warning based on Forge data.

psyche_value

A value or direction the user wants to live toward, such as honesty, courage, or steadiness.

behavior_pattern

A repeating loop across situations, including cue, thought, emotion, action, payoff, and cost.

belief_entry

One explicit belief sentence, such as “If I disappoint people, they will leave me.”

mode_profile

One recurring inner state or role, such as inner critic, detached protector, or healthy adult.

trigger_report

One specific emotionally important episode recorded as what happened, what was felt, what was thought, and what happened next.

Publish Path

npm package first. Public GitHub repo alongside it.

The clean distribution path for Forge today is to publish forge-openclaw-plugin to npm, keep the public Forge repository up to date, and point OpenClaw installs at the npm package.

The docs and screenshots on this page are designed to be the public landing surface for that package and repository.
