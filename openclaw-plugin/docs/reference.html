Runtime Surfaces

The main surfaces Forge exposes.

These are the top-level surfaces you interact with in development and integration work.

Web app

The main UI is served under /forge/. It includes planning, wiki, preferences, Psyche, sleep, sports, and settings.

HTTP API

The versioned API is served under /api/v1/. The health route is /api/v1/health.

Agent adapters

OpenClaw, Hermes, and Codex expose curated Forge operations on top of the same model instead of mirroring the whole internal API.

API Groups

The main API families.

Forge keeps the route groups separate by domain instead of putting all behavior into one generic endpoint family.

Execution

Goals, projects, tasks, habits, task runs, rewards, calendar records, and weekly review live under the versioned app API.

Wiki

Pages, settings, search, health, sync, embeddings, and ingest jobs live under the wiki route family.

Health and Psyche

Sleep, sports, beliefs, values, modes, trigger reports, and related notes stay in their own explicit route groups.

Adapters

How Forge is exposed to agents.

The adapters share the same Forge model, but each package follows the conventions of its host runtime.

OpenClaw

Published plugin package with runtime commands, local startup, health checks, and UI handoff.

Hermes

Python plugin package with the same curated Forge tool surface and a Hermes-native installation path.

Codex

Repo-local plugin surface for shared runtime work inside this repository.

Ownership Model

Humans and bots can share one Forge runtime.

Ownership stays explicit through userId, while links across owners stay valid.

Explicit owners

Records can belong to either a human or a bot. The owner is part of the core model, not hidden adapter metadata.

Scoped reads

Read routes can focus on one owner or several owners through explicit query scope.

Cross-owner links

A human-owned project can still point to bot-owned tasks, strategies, notes, or wiki-linked records.

Repository Packages

Where the main code lives.

Forge is one repository with separate package boundaries for the app, API, docs, and adapters.

Main app

src/ contains the React app and server/ contains the Fastify API, data layer, and runtime services.

Publishable packages

openclaw-plugin/ contains the OpenClaw package and docs site. plugins/forge-hermes/ contains the Hermes package.

Repo-local adapter

plugins/forge-codex/ contains the repo-local Codex adapter surface.

Guides

Longer markdown guides live in docs/ for multi-user strategy, preferences, OpenClaw, and Hermes setup.
